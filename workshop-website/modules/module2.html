Key Concepts

Custom Tools with @tool Decorator

The @tool decorator transforms Python functions into agent-callable tools:

@tool
def custom_function(param: str) -> str:
    """
    Clear description for LLM tool selection.
    
    Args:
        param (str): Description of parameter
        
    Returns:
        str: Description of return value
    """
    return f"Processed: {param}"

Model Context Protocol (MCP)

MCP is an open standard that enables AI agents to connect to external tools and services through a unified interface - think "USB-C for AI agents".

Before MCP vs With MCP

# ‚ùå Before MCP: Custom integration for each service
weather_api = WeatherAPI(api_key="...")
database = DatabaseConnector(host="...")
file_system = FileManager(path="...")

# ‚úÖ With MCP: One standard interface for everything
mcp_client = MCPClient(lambda: stdio_client(...))
with mcp_client:
    tools = mcp_client.list_tools_sync()  # Auto-discover all capabilities
    agent = Agent(tools=tools)  # Ready to use!

Step-by-Step Instructions

Create exercise2-custom-tools.py with custom tool definitions:

from strands import Agent, tool
from strands_tools import calculator, current_time

@tool
def letter_counter(word: str, letter: str) -> int:
    """
    Count occurrences of a specific letter in a word.
    
    Args:
        word (str): The input word to search in
        letter (str): The specific letter to count
        
    Returns:
        int: The number of occurrences of the letter in the word
    """
    # Input validation
    if not isinstance(word, str) or not isinstance(letter, str):
        return 0
    if len(letter) != 1:
        raise ValueError("The 'letter' parameter must be a single character")
    
    return word.lower().count(letter.lower())

@tool
def text_reverser(text: str) -> str:
    """
    Reverse the order of characters in text.
    
    Args:
        text (str): Text to reverse
        
    Returns:
        str: Reversed text
    """
    if not isinstance(text, str):
        return "Error: Input must be a string"
    
    return text[::-1]

Combine built-in and custom tools in a single agent:

# Create agent with both built-in and custom tools

    calculator,           # Built-in tool

Advanced Code Examples

Tool Best Practices

@tool
def word_counter(text: str) -> int:
    """
    Count the number of words in text.
    
    Args:
        text (str): Text to count words in
        
    Returns:
        int: Number of words
    """
    # Input validation
    if not isinstance(text, str):
        return 0
    
    # Handle edge cases
    if not text.strip():
        return 0
    
    # Core functionality
    words = text.strip().split()
    return len(words)

MCP Integration (Optional Advanced Exercise)

Critical MCP Pattern: Context Manager Usage

The most important concept in MCP integration is proper context manager usage:

# ‚ùå WRONG WAY (causes MCPClientInitializationError):
client = MCPClient(...)
tools = client.list_tools_sync()  # Outside context manager
agent = Agent(tools=tools)        # Outside context manager
with client:
    response = agent("task")       # Agent created outside, used inside

# ‚úÖ CORRECT WAY (works properly):
client = MCPClient(...)
with client:                      # Context manager first
    tools = client.list_tools_sync()  # Inside context manager
    agent = Agent(tools=tools)        # Inside context manager
    response = agent("task")           # Everything inside

MCP Filesystem Integration

from mcp import stdio_client, StdioServerParameters
from strands.tools.mcp import MCPClient
from pathlib import Path

# Create MCPClient for filesystem server
current_dir = str(Path.cwd())
client = MCPClient(lambda: stdio_client(
    StdioServerParameters(
        command="npx",
        args=["-y", "@modelcontextprotocol/server-filesystem", current_dir]
    )
))

# CRITICAL: Use agent WITHIN the client context manager
with client:
    # Get tools and create agent INSIDE the context manager
    tools = client.list_tools_sync()
    
    agent = Agent(
        model=BedrockModel(
            model_id="us.anthropic.claude-3-7-sonnet-20250219-v1:0",
            temperature=0.3
        ),
        tools=tools,
        system_prompt="You are a filesystem assistant with file operation capabilities."
    )
    
    # Now you can use the agent with MCP tools
    response = agent("List all .py files in the current directory")
    print(response)

Common Issues & Solutions

Issue: Tool Not Being Called

Agent doesn't use custom tool when expected

Solution: Improve tool docstring and type hints:

@tool
def better_tool(input_text: str) -> str:
    """
    CLEAR, SPECIFIC description of what this tool does.
    Be explicit about when the LLM should use this tool.
    
    Args:
        input_text (str): Detailed description of expected input
        
    Returns:
        str: Detailed description of what gets returned
    """
    return process_input(input_text)

Issue: MCPClientInitializationError

MCPClientInitializationError: Agent created outside context manager

Solution: Always create agents inside the MCP context manager:

# ‚úÖ CORRECT pattern
with mcp_client:
    tools = mcp_client.list_tools_sync()
    agent = Agent(tools=tools)  # Create agent inside context
    response = agent("task")

Issue: MCP Server Not Found

Error: MCP server not working

Solution: Install MCP servers globally:

# Install Node.js first, then:
npm install -g @modelcontextprotocol/server-filesystem
npm install -g @modelcontextprotocol/server-memory

# Verify installation
npx @modelcontextprotocol/server-filesystem --help

Issue: Tool Parameter Validation

TypeError: Tool received unexpected parameter type

Solution: Add comprehensive input validation:

@tool
def safe_tool(input_data: str) -> str:
    """Tool with comprehensive error handling."""
    try:
        # Input validation
        if not isinstance(input_data, str):
            return "Error: Input must be a string"
        
        if not input_data.strip():
            return "Error: Input cannot be empty"
        
        # Tool logic
        result = process_data(input_data)
        return result
        
    except ValueError as e:
        return f"Input validation error: {e}"
    except Exception as e:
        return f"Tool execution error: {e}"
