Zero-Setup Workshop Environment

Get started in under 2 minutes with a fully configured cloud development environment. No local installation required!

1

Fork the Repository

Click the button below to fork the workshop repository to your GitHub account:

Fork Repository
2

Launch Codespace

After forking, click the green "Code" button and select "Create codespace on main":

# Or use the direct link after forking:
# https://github.com/YOUR_USERNAME/strands-workshop

Alternative: Use the button below after forking:

Open in Codespaces
3

Wait for Environment Setup

Codespaces will automatically:

  • ‚úÖ Install Python 3.10+ and all dependencies
  • ‚úÖ Set up AWS CLI and Docker
  • ‚úÖ Install Strands SDK and all required packages
  • ‚úÖ Pull MCP Docker images for advanced tools
  • ‚úÖ Configure Q CLI for Amazon Q integration
  • ‚úÖ Set up MCP Inspector UI for debugging

This process takes 2-3 minutes. You'll see the setup progress in the terminal.

4

Configure AWS Credentials

Once the environment is ready, configure your AWS credentials:

aws configure

Then enable Claude 3.7 Sonnet in AWS Bedrock Console

5

Verify Setup & Start Workshop

# Verify everything is working
python setup/verify_setup.py

# Start with Module 1
cd exercises/module1-basics
python exercise1-hello-agent.py

Local Environment Setup

Skip this section if you're using GitHub Codespaces (recommended above).

Repository Setup

# Fork and clone your repository
git clone https://github.com/YOUR_USERNAME/strands-workshop.git
cd strands-workshop

Virtual Environment

# Create virtual environment
python -m venv .venv

# Activate on macOS/Linux
source .venv/bin/activate

# Activate on Windows
.venv\Scripts\activate

Install Dependencies

# Install all dependencies
pip install -r requirements.txt

AWS Configuration

Configure AWS CLI

aws configure

You will be prompted to enter:

  • AWS Access Key ID
  • AWS Secret Access Key
  • Default region name (recommended: us-west-2)
  • Default output format (json)

Enable Claude 3.7 Sonnet in AWS Bedrock

  1. Go to the AWS Bedrock Console
  2. Click on "Model access"
  3. Find "Claude 3.7 Sonnet" and click "Edit"
  4. Toggle the access switch to "On"
  5. Click "Save changes"

API Keys (Optional)

Environment Variables

Create a .env file in the workshop root directory:

# AWS (configured via aws configure)
AWS_REGION=us-east-1

# Optional API Keys
OPENAI_API_KEY=sk-...
NVIDIA_API_KEY=nvapi-...
OPENROUTER_API_KEY=sk-or-...

API Key Sources

  • OpenAI API Keys
  • NVIDIA NIM API Keys
  • OpenRouter API Keys
